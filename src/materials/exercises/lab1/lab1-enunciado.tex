% ---- Learning Objectives ----------------------------------------------------
\section{Learning Objectives}

After completing this lab you will be able to:

\begin{itemize}[leftmargin=1.5em]
  \item Perform basic active reconnaissance on a target host using standard tools
  \item Identify exposed services and gather information from HTTP responses
  \item Use discovered credentials to obtain shell access via SSH
  \item Navigate a multi-hop network environment (firewall, router, server, workstation)
  \item Document findings in a structured penetration testing report
\end{itemize}

% ---- Scenario ---------------------------------------------------------------
\section{Scenario}

You are a junior penetration tester hired to assess the perimeter of
\textbf{PEBCAK Corp} -- a fictional company whose name stands for
\textit{Problem Exists Between Chair And Keyboard}.

Your point of entry is the \textbf{Attacker} node in the lab environment.
The target company exposes a public hostname: \texttt{lab1}.
Your goal is to enumerate the target, find exposed information, and pivot
as deep as possible into the internal network.

\begin{mdframed}[backgroundcolor=labgray, linecolor=labred, linewidth=1pt]
\textbf{Rules of engagement}
\begin{itemize}[leftmargin=1.5em, topsep=2pt]
  \item Only attack nodes within the lab environment
  \item Do not modify pfSense firewall configuration
  \item Document every step -- commands run, output received, decisions made
  \item If the lab breaks, notify your instructor (do not attempt to fix infrastructure)
\end{itemize}
\end{mdframed}

% ---- Tasks ------------------------------------------------------------------
\section{Tasks}

Complete the following tasks in order. Document each step.

\subsection{Task 1 -- Web Reconnaissance}

\begin{enumerate}[leftmargin=2em]
  \item Open a browser on the Attacker node and navigate to \texttt{http://lab1}
  \item Identify the company name displayed on the page
  \item View the page source (\texttt{Ctrl+U} or \texttt{curl -s http://lab1})
  \item Look for any hidden information or comments in the HTML
\end{enumerate}

\textit{Deliverable: What hint did you find in the page source?}

\subsection{Task 2 -- Service Discovery}

\begin{enumerate}[leftmargin=2em]
  \item Using the hint from Task 1, find an additional resource on the web server
  \item What credentials are exposed at that resource?
  \item What service do the credentials grant access to?
\end{enumerate}

\textit{Deliverable: Service type, host, username.}

\subsection{Task 3 -- Initial Access (Flag 1)}

\begin{enumerate}[leftmargin=2em]
  \item Use the discovered credentials to connect to the target service
  \item Confirm you have a shell on the target server
  \item List the contents of the home directory and read the flag file
\end{enumerate}

\textit{Deliverable: Contents of the flag file. Read it carefully -- it contains more than just the flag.}

\subsection{Task 4 -- Pivot to Firewall (Flag 2)}

\begin{enumerate}[leftmargin=2em]
  \item Using information found in Task 3, connect to the internal firewall
  \item Read the flag file in the administrator's home directory
\end{enumerate}

\textit{Deliverable: Flag string. What security mistake made this possible?}

\subsection{Task 5 -- Reach the Workstation (Flag 3)}

\begin{enumerate}[leftmargin=2em]
  \item The firewall flag reveals a further internal host
  \item From the server, connect to the internal workstation
  \item Read the flag in the user's home directory
\end{enumerate}

\textit{Deliverable: Flag string. How many hops from the Attacker to the workstation?}

% ---- Deliverables -----------------------------------------------------------
\section{Deliverables}

Submit a brief report (1--2 pages) covering:

\begin{itemize}[leftmargin=1.5em]
  \item \textbf{Reconnaissance summary:} what did you find and how
  \item \textbf{Access path:} step-by-step from Attacker to each target
  \item \textbf{All 3 flags captured:} paste the exact flag strings
  \item \textbf{Reflection:} one thing you would fix as the sysadmin to stop this attack
\end{itemize}

% ---- Hints ------------------------------------------------------------------
\section{Hints}

\begin{mdframed}[backgroundcolor=labgray, linecolor=labred, linewidth=0.5pt]
\textbf{Stuck?} Hints are ordered by how much they reveal. Try each before moving to the next.

\begin{enumerate}[leftmargin=2em, topsep=4pt]
  \item Web servers often have more than one page. Think about what the company name suggests.
  \item \texttt{curl} and your browser both let you read page source. Comments in HTML are
        written between \texttt{<!--} and \texttt{-->}.
  \item Once you have SSH credentials, the command is: \texttt{ssh user@hostname}
  \item Read the flag file \textit{completely} -- it often contains more than the flag itself.
  \item From a server inside a network, you can reach other internal hosts with \texttt{ssh user@ip}.
\end{enumerate}
\end{mdframed}
