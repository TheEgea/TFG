\section{Step 1 — Landing Page \& Source Inspection}

Open Firefox on the Attacker and navigate to \texttt{http://lab1}.

\begin{lstlisting}
$ curl -s http://lab1
<!DOCTYPE html>
<html lang="en">
<head><meta charset="UTF-8"><title>PEBCAK Corp</title></head>
<body>
  <h1>PEBCAK CORP</h1>
  <p class="subtitle">Problem Exists Between Chair And Keyboard</p>
  <!-- sometimes simplify and search -->

</html>
\end{lstlisting}

\textbf{Finding:} HTML comment \texttt{<!-- sometimes simplify and search -->}

\textbf{Interpretation:} "simplify and search" suggests looking for a shorter or
simpler path. The company name is PEBCAK — try \texttt{/pebcak} or \texttt{/pebcak.html}.

% ── Step 4 ───────────────────────────────────────────────────────────────────
\section{Step 4 — Lateral Movement to Firewall}

From the Server, the student uses the credentials found in \texttt{flag.txt}
to access the firewall's management interface directly on the internal LAN:

\begin{lstlisting}
blackmesa@pebcak:~$ ssh admin@172.16.0.1
admin@172.16.0.1's password: pfsense

[2.7.2-RELEASE][admin@pfSense.localdomain]/root:
\end{lstlisting}

\textbf{Finding:} pfSense shell access from the compromised server.

The route exists because the VyOS router forwards traffic between the
Servers LAN (192.168.20.0/24) and the pfSense LAN interface (172.16.0.1/30).

% ── Common mistakes ──────────────────────────────────────────────────────────
\section{Common Student Mistakes}

\begin{itemize}[leftmargin=1.5em]
  \item \textbf{Using IP instead of hostname:} \texttt{ssh blackmesa@192.168.0.29}
        works but students should understand the DNS resolution chain.
  \item \textbf{Wrong password quoting:} The \texttt{!} in \texttt{!Bl4kM3s\$}
        is special in bash. It works fine when typed interactively; issues only
        arise when scripting.
  \item \textbf{Modifying pfSense:} Some students attempt to explore the pfSense
        web UI. Remind them the scope is the Server, not the infrastructure.
  \item \textbf{Not reading flag.txt fully:} The second half of the file
        (pfSense credentials) is needed for Step 4. Many students stop at the flag.
\end{itemize}
