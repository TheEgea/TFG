\chapter{Objectives and Scope}
%\section{Main Objective}

Develop, implement, and validate a reusable teaching package of practical cybersecurity labs based on EVE-NG\cite{eve_ng}, enabling students in the "Introduction to Cybersecurity" course to acquire practical skills in ethical pentesting through realistic and automated scenarios.
\section{Specific Objectives (Measurable with KPIs)}

\subsection{OE1: Design and Implement EVE-NG Labs}


\textbf{Description}: Create four functional and interconnected thematic labs.
\textbf{Measurable KPIs}:
\begin{itemize}
\item 4 fully functional .unl topologies (100\%)
\item Deployment time $\leq$ 2 minutes per lab (95\% of cases)
\item VM boot success rate $\geq$ 95\%
\item Compatibility with EVE-NG Community and Professional editions
\end{itemize}

\textbf{Deliverables}:
\begin{itemize}
\item 4 .unl topology files (.unl), one per lab
\item 5 core virtual machine images (VyOS, pfSense~CE, Ubuntu Server~24.04, Ubuntu Desktop~24.04, Parrot~OS~Security), sized and optimised for concurrent EVE-NG deployment on constrained HomeLab hardware
\item Technical configuration documentation for each lab
\end{itemize}

\subsection{OE2: Develop Automation Scripts}

\textbf{Description}: Develop utility scripts that automate recurring lab management tasks, reducing manual effort for instructors and ensuring repeatable configurations across lab sessions.

\textbf{Measurable KPIs}:
\begin{itemize}
\item At least one functional automation script per lab covering a recurring task
\item Scripts documented and reusable across different EVE-NG installations
\item All scripts tested and verified in the project's EVE-NG environment
\end{itemize}

\textbf{Deliverables}:
\begin{itemize}
\item Bash/Python utility scripts for common tasks (e.g.\ bridge configuration,
      ISO upload, node connectivity checks)
\item Scripts documented with usage instructions in the repository
\end{itemize}

\subsection{OE3: Develop Structured Teaching Material}

\textbf{Description}: Develop comprehensive documentation for students and teachers.

\textbf{Measurable KPIs}:
\begin{itemize}
\item 4 student lab guides (1 per lab) in PDF format --- task description,
      hints, and tool reference
\item 4 instructor resolution guides (1 per lab) in PDF format --- step-by-step
      walkthrough, expected output, and assessment rubric
\item Assessment rubric included in each resolution guide with quantifiable criteria
\item Lab setup procedure documented for instructors in each resolution guide
\end{itemize}

\textbf{Deliverables}:
\begin{itemize}
\item Student lab guide PDF (enunciado) per lab --- task description, objectives,
      hints, and tools reference
\item Instructor resolution guide PDF (resolució) per lab --- full walkthrough,
      flag values, rubric (100~pts), and common errors
\item Technical configuration reference per lab in the repository
\end{itemize}

\subsection{OE4: Validate Usability and Educational Effectiveness}

\textbf{Description}: Verify that the package meets educational and technical requirements through a reduced pilot group representative of the target student population.

\textbf{Measurable KPIs}:
\begin{itemize}
\item Average lab resolution time within the 90--120 minute target window
\item All pilot participants able to complete the lab with the provided guide
\item User satisfaction score $\geq$ 4/5 (post-session survey)
\item Zero blocking technical incidents during pilot sessions
\end{itemize}

\textbf{Deliverables}:
\begin{itemize}
\item Pilot session report with timing and usability observations
\item Post-session survey results (satisfaction, difficulty, clarity)
\item List of improvements implemented based on pilot feedback
\end{itemize}

\textbf{Note}: The pilot group comprises 4--5 individuals from the GEISI student
population, potentially including participants with documented learning
difficulties (e.g.\ dyslexia), to assess the effectiveness of the
accessibility measures incorporated into the lab documentation.

\section{Definition of the Client and Final User}

The package targets two distinct groups: a principal client (the responsible professor) and end users (students). Secondary audiences may extend the material beyond the original course.

\subsection{Principal Client}
\begin{itemize}
\item \textbf{Responsible professor}: Pere Vidiella i Catalan
\item \textbf{Course}: Introduction to Cybersecurity (GEISI, Tecnocampus)
\item \textbf{Context}: Undergraduate teaching in practical cybersecurity
\end{itemize}

\subsection{Primary End Users}

\begin{itemize}
\item \textbf{GEISI Students}: 25-30 students per edition
\item \textbf{Profile}: Basic knowledge of networks and operating systems
\item \textbf{Objective}: Learn ethical pentesting in a practical and safe manner
\end{itemize}

\subsection{Secondary End Users}

\begin{itemize}
\item \textbf{Cybersecurity Faculty}: Other teachers interested in reusing the material
\item \textbf{Postgraduate Students}: Possible extensions of the material for advanced levels
\end{itemize}

\subsection{Potential Audience}

Beyond the primary users, the material is designed for reuse across different educational contexts.

\subsubsection{Internal Scope (Tecnocampus)}

\begin{itemize}
\item Other courses related to cybersecurity
\item Research projects in computer security
\item Continuing education and professional certifications
\end{itemize}

\subsubsection{External Scope}

\begin{itemize}
\item Educational institutions with training in cybersecurity
\item Specialized vocational training centers
\item Companies with internal training programs in security
\end{itemize}

\section{Scope Boundaries}

\textbf{Included in this bachelor's thesis:}

\begin{itemize}
    \item Design and implementation of 4 pentesting labs in EVE-NG
    \item Integration of real vulnerabilities and techniques aligned with 
          OWASP Top 10\cite{owasp2021} and CIS Critical Controls\cite{cis_controls}
    \item Utility automation scripts for recurring lab management tasks (e.g.\ bridge configuration, ISO upload, connectivity checks)
    \item Complete technical documentation for instructors and future maintainers
    \item Student guides and learning objectives for each lab
    \item Assessment rubrics for evaluating student lab completion
\end{itemize}

\textbf{Explicitly NOT included (out of scope):}

\begin{itemize}
    \item Integration with third-party Learning Management Systems (LMS) or Moodle
    \item Development of proprietary lab management software or web interfaces
    \item Creation of automated grading systems beyond basic script validation
    \item Commercial licensing or distribution models
    \item Formal pedagogical evaluation studies 
          (e.g., pre/post learning assessments)
    \item Creation of labs beyond the four specified domains
\end{itemize}
