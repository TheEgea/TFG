\section*{Lab Startup Checklist}

\begin{enumerate}
    \item Start all nodes from the EVE-NG web interface in order:
          pfSense $\to$ VyOS $\to$ Server $\to$ PC1 $\to$ Parrot.
    \item Run the bridge script on the EVE-NG host (lost on reboot; persistent via
          \texttt{udev} rule):
\begin{lstlisting}[language=bash]
ip addr add 192.168.20.x/24 dev vnet0_2
ip addr add 192.168.10.x/24 dev vnet0_3
ip route add 172.16.0.0/30 via 192.168.20.1
\end{lstlisting}
    \item Verify DNS resolution from Parrot: the DNS server on Parrot must point to
          pfSense's WAN IP so that \texttt{lab1} resolves.
\begin{lstlisting}[language=bash]
# On Parrot:
curl http://lab1          # Should return PEBCAK Corp HTML
\end{lstlisting}
    \item Confirm flag is in place on the Server node:
\begin{lstlisting}[language=bash]
sshpass -p '!Bl4kM3s$' ssh blackmesa@192.168.20.x cat ~/flag.txt
# Expected: FLAG{p3bc4k_s3rv3r_0wn3d}
\end{lstlisting}
\end{enumerate}
