\chapter{Introduction and Context}

\section{Motivation and Justification}

Ethical hacking, or pentesting, is a critical skill in today's cybersecurity landscape. With the increasing

reliance on virtualized environments, there is a growing need for practical training platforms that simulate

real-world scenarios. While platforms such as HackTheBox and TryHackMe have become increasingly popular for

cybersecurity training, they often focus on isolated challenges and gamified scenarios that may not directly

reflect the practical application of computer science fundamentals taught in a degree program. EVE-NG

(Emulated Virtual Environment Next Generation) offers a complementary and more customizable approach,

providing a robust solution for creating controlled, hands-on learning environments.

This project aims to develop a series of practical laboratories using EVE-NG that explicitly bridge the

foundational concepts covered throughout the degree program---including databases, network protocols, and

operating system internals---with real-world security vulnerabilities and pentesting techniques. By

demonstrating how these academic concepts can be exploited, students gain a deeper appreciation for

defensive security practices and the critical importance of secure system design. This approach transforms

abstract knowledge into hands-on experience where students can observe the direct consequences of security

misconfigurations and understand why rigorous cybersecurity practices are essential.

Furthermore, by leveraging EVE-NG, this project seeks to fill a gap in comprehensive educational resources

that integrate multiple pentesting tools and attack scenarios into a cohesive learning environment aligned

with the Computer Engineering curriculum. This reproducible, risk-free, and fully customizable approach

provides students with the opportunity to practice ethical hacking in a controlled setting, progressively

building upon prior learning while preparing them for the professional responsibility of securing the

systems they will design and manage in their careers.

\chapter{Object of the Project}

\section{Problem Statement and Educational Gap}

Cybersecurity education faces a critical disconnect: students learn theoretical 
IT foundations---databases, network protocols, operating systems---yet rarely 
experience how design flaws translate to security vulnerabilities. Current 
training platforms (HackTheBox, TryHackMe) focus on isolated challenges that 
abstract system complexity. Professional pentesting certifications expect 
hands-on experience in realistic environments, yet academic programs lack such 
environments due to cost, complexity, and vendor lock-in.

Within TecnoCampus's Bachelor's Degree in Computer Engineering, the 
``Introduction to Cybersecurity'' course lacks a structured, reusable, 
reproducible lab environment that: (1) aligns pentesting with degree learning 
outcomes, (2) demonstrates vulnerabilities in integrated systems, (3) enables 
consistent assessment, (4) reduces instructor burden through automation, and 
(5) remains cost-effective and institution-independent.

\section{Project Description and Scope}

This TFG develops a comprehensive teaching package of four virtualized 
cybersecurity laboratories using EVE-NG for the ``Introduction to Cybersecurity'' 
course. Detailed architecture and design rationale are provided in Appendix A.

\textbf{High-level deliverables:}

\begin{itemize}
\item 4 functional EVE-NG lab topologies (.unl files)
\item 8-12 customized VM images with preconfigured vulnerabilities
\item Automated deployment and reset scripts
\item Complete technical documentation (50-100 pages)
\item Student learning guides with objectives and assessment rubrics
\item Instructor materials including grading rubrics and troubleshooting
\end{itemize}

\textbf{Scope boundaries:} See Appendix A.1 for detailed inclusion/exclusion list.

\section{Why EVE-NG? Selection Rationale}

EVE-NG was selected through comparative analysis of alternatives 
(HackTheBox, TryHackMe, Cisco Netacad, GNS3, SEED Labs). 
Key advantages: full network topology customization, open-source, 
locally deployable, vendor-independent, excellent automation. 
Detailed comparison is provided in Appendix A.2.

\section{Alignment with Computer Engineering Degree}

This project creates explicit connections between four core competencies 
taught throughout the degree and their cybersecurity applications: 
(1) programming and secure coding, (2) software engineering architectures, 
(3) systems and information engineering, (4) computer architecture and networks. 
See Appendix A.3 for detailed mapping.

\section{Justification and Professional Context}

Cybersecurity professionals must recognize vulnerabilities in complex, 
integrated systems using industry standards (OWASP, CIS, NIST) and 
standard tools (Kali, Metasploit, Wireshark). This project prepares students 
to meet these expectations. Full pedagogical and professional justification 
is documented in Appendix A.4.

\section{Expected Outcomes and Success Criteria}

Four functional labs, 8-12 optimised VMs, deployment automation scripts, 
technical documentation, student guides, assessment materials, and a 
reproducible teaching package suitable for multiple course offerings and 
institutional sharing. Success measured by: functionality, documentation 
completeness, student usability, institutional sustainability, and 
transferability. Detailed criteria in Appendix A.5.
